# Booking API Tests

Коллекция содержит базовые запросы и тесты для проверки REST API системы бронирования мероприятий.

## Используемые переменные

В коллекции созданы следующие переменные окружения:

| Переменная       | Описание переменной                          |
|------------------|-----------------------------------------------|
| `{{baseUrl}}`    | URL mock server                       | 
| `{{sessionId}}`  | Токен | 
| `{{bookingId}}`  | `Id` бронирования |

## Структура запросов и тестов


| Запрос                              | Метод | Путь                        | Что проверяется в тестах                                                                 | 
|-------------------------------------|-------|--------------------------------------|--------------------------------------------------------------------------------------------------|
| Авторизация (Login)                 | POST  | `/api/auth/login`      | • Статус 200<br>• Наличие ключа `sessionId` <br>• Значение токена не пустое <br>• Устанавается токен в переменную окружения  |
| Список мероприятий     | GET   | `/api/events`          | • Статус 200<br>• Ответ — массив<br>• Проверка структуры JSON на наличие ключевых полей (id, title, date, …) и типы данных<br> |
| Создание брони        | POST  | `/api/bookings`  | • статус 200 1<br>• наличие `bookingId`<br>• статус 400 при ошибке           |
| Оплата    | POST  | `/api/payments`           | • статус 200 → проверка `status: "PAID"` / `"success"`<br>• статус 400 при ошибке |
| Получение деталей брони             | GET   | `/api/bookings/{{bookingId}}`        | • статус 200<br>• проверка наличия ключевых полей (id, status, tickets, …) и типы данных                |
